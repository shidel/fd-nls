Mise en réseau de FreeDOS - les autres kernels TCP/IP

  Certains autres noyaux TCP/IP externes peuvent aussi être légalement
  téléchargés et utilisés.
    * Novell distribue sa pile de protocole TCP/IP 16 bits populaire
      "TCP16.EXE" (voyez : http://bbs.actapricot.org/files/area37/
      tcp16.exe (*01)) qui s'exécute directement sur le pilote ODI.
      Quelques programmes comme Josh's "Tiny" remote control software
      c'est-à-dire le petit logiciel de contrôle à distance de Josh
      (cf. http://josh.com/tiny/ (*02)) et "les clients NFS pour DOS"
      de Tsoft (cf. http://www.rawbandwidth.com/software/ (*03))
      utilisent toujours ce noyau.
    * Le "client MS 3.0" de Microsoft (cf. ftp://ftp.microsoft.com/
      bussys/Clients/MSCLIENT/ (*04)) comprend son propre kernel TCP/IP
      "TCPTSR.EXE".
      Il peut aussi être utilisé par des programmes externes. Il peut
      être utile cependant de vérifier d'abord les besoins en mémoire.
      Le noyau TCP/IP du client MS semble consommer beaucoup de mémoire
      DOS conventionnelle.
  Veuillez visiter Microsoft et Novell pour davantage d'informations.

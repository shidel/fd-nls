Command: lastdrivehigh

  Lastdrivehigh définit la dernière lettre à laquelle un lecteur peut
  être affecté.
  Lastdrivehigh est une commande CONFIG.SYS / FDCONFIG.SYS.

Syntaxe :

  lastdrivehigh=x
    où x est la dernière lettre qui sera disponible.

Options :

  - aucune -

Commentaires :

  Par défaut, FreeDOS définira x de manière suffisamment haute pour tous
  les lecteurs de disquettes détectables et les partitions de disque
  dur, plus un pour un CD ou un autre lecteur. Vous pouvez vouloir
  définir x plus haut si vous souhaitez utiliser plus d'un lecteur que
  FreeDOS ne peut détecter automatiquement (lecteurs CD ou zip, etc.).
  Régler x sur une valeur plus haute que nécessaire gaspillera une
  petite quantité de mémoire.
  LASTDRIVEHIGH est similaire à LASTDRIVE, sauf qu'il tente de se
  charger dans l'espace mémoire haute d'abord (si cela ne marche pas,
  il devrait se charger dans la mémoire conventionnelle).
  Remarque : l'ordre avec lequel vous chargez les périphériques peut
  avoir un grand impact sur la quantité de mémoire libre disponible.
  En général essayez de charger les gros (en termes d'utilisation
  mémoire) programmes d'abord dans la mémoire haute.
  Important : vous devriez avoir un gestionnaire de mémoire haute
  tel que (HIMEM) ou HIMEMX ou bien JEMMEX installé (device=himem.exe
  ou device=himemx.exe avant dans CONFIG.SYS / FDCONFIG.SYS) avant
  d'utiliser cette option.
  LASTDRIVEHIGH est une commande interne à kernel.sys et ne nécessite
  aucun autre fichier pour fonctionner.

Exemples :

  DANS CONFIG.SYS / FDCONFIG.SYS:
    lastdrivehigh=z  active les lettres de lecteur de A à Z.

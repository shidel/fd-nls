Commande : krnl-dvl (kernel de développement)

  Le KERNEL ou noyau (KERNEL.SYS) est le coeur de FreeDOS.
  FreeDOS a un KERNEL STABLE et un de développement (instable).

Syntaxe: 

  - aucune -

Options :

  - aucune -

Comments:

  Le KERNEL est nécessaire pour charger les pilotes via CONFIG.SYS /
  FDCONFIG.SYS et pour faire fonctionner COMMAND.COM ou d'autres
  shells dans FreeDOS. Alors que le KERNEL charge CONFIG.SYS /
  FDCONFIG.SYS, le SHELL (par ex : COMMAND.COM) charge AUTOEXEC.BAT.
  Pour installer le KERNEL sur une disquette ou un DD vous devez
  utiliser la commande SYS. SYS copie le KERNEL à un endroit
  spécial sur média de boot ; sinon il n'est pas trouvé lors du
  démarrage et le système ne fonctionnera pas. Pour cette raison
  KERNEL.SYS doit TOUJOURS être dans le répertoire racine du média
  d'amorçage.
  The KERNEL de développement (instable) prend en charge COUNTRY.SYS
  et CHCP et possède une prise en charge expérimentale de Win386 /
  WfW 3.11.
  Pour des besoins standards, veuillez utiliser le KERNEL stable.

Exemples :

  - aucun -
